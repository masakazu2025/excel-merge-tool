# REQ-006 エラーハンドリング

## 概要

バックエンド・フロントエンド双方で、処理が止まることなく適切なエラー情報を返す・表示する。

## 方針

- バックエンドは常に構造化されたエラーレスポンスを返す（例外をそのまま伝播させない）
- フロントエンドは fetch の成否・レスポンスの構造を必ず検証し、クラッシュを防ぐ
- エラーコードは `E001〜E007`（バックエンド）/ `F001〜F003`（フロントエンド）で管理する
- 想定外のエラーはキャッチオール（E005 / F003）で吸収する

## エラーレスポンス形式（バックエンド）

```json
{
  "error_code": "E002",
  "message": "ファイルが破損しているか読み込めません"
}
```

HTTP ステータスコード:

| コード | ステータス |
|--------|-----------|
| E001〜E005 | 422 Unprocessable Entity |
| E006〜E007 | 404 Not Found |

## 受け入れ条件

- [ ] バックエンドの全エンドポイントで未ハンドル例外が外に漏れない
- [ ] フロントエンドでネットワークエラー時にクラッシュしない
- [ ] フロントエンドでAPIエラー時にエラーメッセージを表示する
- [ ] フロントエンドで不正なJSONを受け取ってもクラッシュしない
