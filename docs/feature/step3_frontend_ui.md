# Step 3: フロントエンド実装（React + Vite + Tailwind）

## 概要

Vite + React 18 + TypeScript + Tailwind CSS で構築するSPAアプリ。
FastAPIが静的配信する（`frontend/dist/` をビルド成果物として出力）。

---

## 技術スタック

| 項目 | 採用技術 |
|------|----------|
| フレームワーク | React 18 + TypeScript |
| ビルドツール | Vite |
| CSSフレームワーク | Tailwind CSS v3 |
| ルーティング | React Router v6 |
| HTTPクライアント | fetch API（標準） |

---

## 画面構成

### ① ホーム画面 `/`

```
┌──────────────────────────────────────────────────┐
│  Excel 差分比較ツール                             │
├──────────────────────────────────────────────────┤
│  ┌──────────────┐                               │
│  │ base.xlsx    │ [ファイル選択]                  │
│  │ file_b.xlsx  │ [ファイル選択]                  │
│  │ file_c.xlsx  │ [ファイル選択]                  │
│  └──────────────┘                               │
│  [比較実行]                                       │
├──────────────────────────────────────────────────┤
│  過去のレポート                                   │
│  ▼ 2026-03-14 15:30  base.xlsx vs ...  (42件)   │
│    2026-03-10 09:12  base.xlsx vs ...  (15件)   │
└──────────────────────────────────────────────────┘
```

- Excelファイル3つをアップロード → `POST /api/compare`
- 比較中はローディングスピナーを表示
- 完了後、自動でレポート画面へ遷移
- 過去レポート一覧から選択して再表示可能

## 機能仕様

### シートタブ

- シート名と差分件数を表示（例: `Sheet1 (3)`）
- 競合が存在するシートには警告アイコン（⚠）を表示

### セル差分テーブル

| カラム | 内容 |
|--------|------|
| チェックボックス | 選択（将来のCSV出力用。現フェーズでは参考表示） |
| セル位置 | 例: `B4` |
| 種別 | `値` / `数式` / `リッチテキスト` / `コメント` / `日付` |
| 現在値（base） | ベースファイルの値 |
| Bの値 | ファイルBの値（変更なしは `-`） |
| Cの値 | ファイルCの値（変更なしは `-`） |
| 状態 | diff_hint の表示（新規 / 追記 / 削除 / 変更 / 日付更新 / ⚠競合） |

### 競合セルのUI

- `conflict: true` の行はチェック時にラジオボタンが展開
- B採用 / C採用 を明示的に選ばないとCSV出力できない（将来実装）

### フィルター

| フィルター | 条件 |
|-----------|------|
| 全て | 全差分を表示 |
| 競合のみ | `conflict: true` |
| Bの変更 | `b_value != null && !conflict` |
| Cの変更 | `c_value != null && !conflict` |
| 要確認 | `review_required: true` |

### 図形・画像・グラフセクション

- 折りたたみ可能（デフォルト展開）
- 追加・削除のみ検出対象（画像・グラフの内容変更は非対応）
- テキストボックスはテキスト変更も表示

## 決定事項

| 項目 | 決定内容 |
|------|----------|
| CSSフレームワーク | Tailwind CSS v3 |
| ビルドツール | Vite（開発サーバー + 本番ビルド） |
| 言語 | TypeScript |
| JSX/Babel | Viteが処理するため不要（開発時のみ） |
| 図形の表示 | セルテーブルとは別セクションに分離。折りたたみ可能。 |
| モバイル対応 | 不要（PC業務環境前提） |

## 懸念・未解決事項

### 【中】大量差分時のレンダリング性能

差分が数千行ある場合、全行をDOMにレンダリングするとブラウザが重くなる可能性がある。
まず全件レンダリングで実装し、問題が発生した場合にページネーションで対応する。
Vite環境なので仮想スクロールライブラリ（`@tanstack/virtual`等）も導入可能。
