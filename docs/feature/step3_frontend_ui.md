# Step 3: フロントエンド実装（React UI・CSV出力）

## 概要

`report.html` をブラウザで開き、差分を確認・選択してマージ指示書CSVをダウンロードする。
HTMLファイル内のJavaScript（React）として実装する。JSX・Babelは不使用。`React.createElement()` で直接記述する。

---

## 入力

`<script type="application/json" id="diff-data">` タグから取得した差分JSON（Step 2が生成）

```js
const diffData = JSON.parse(document.getElementById('diff-data').textContent);
```

## 出力

`merge_commands.csv` — マージ実行指示書（Step 4のVBAが読み込む）

## 機能仕様

### シートタブ

- シート名と差分件数（例: `Sheet1 (3)`）を表示
- 競合が存在するシートには警告アイコンを表示

### セル差分テーブル

| カラム | 内容 |
|--------|------|
| チェックボックス | マージ対象として選択する |
| セル位置 | 例: `B4` |
| 種別 | `値` / `数式` / `リッチテキスト` / `コメント` |
| マージ元の現在値 | ベースファイルの値 |
| Bの値 | ファイルBの値（変更なしの場合は `-`） |
| Cの値 | ファイルCの値（変更なしの場合は `-`） |
| ステータス | 競合（B≠C）の場合は `⚠ 競合` を表示 |

### 図形・画像・グラフセクション（別セクション）

セル差分テーブルとは**分離した折りたたみ可能セクション**として表示する。

| カラム | 内容 |
|--------|------|
| チェックボックス | マージ対象として選択する |
| 図形名/ID | `Shape.Name` または内部ID |
| 種別 | `テキストボックス` / `画像` / `グラフ` / `その他図形` |
| 状態 | `B追加` / `C追加` / `B削除` / `C削除` / `テキスト変更`（テキストボックスのみ） |
| ソース | 採用するファイル（B or C） |

- 図形の**追加・削除**のみ検出対象。画像・グラフの内容変更は検出しない（Step 1 制限事項）
- テキストボックスはテキスト内容の変更も検出対象

### 競合セルの選択UI

B・Cが同じセルを異なる値に変更している場合（`conflict: true`）:
- チェックボックス選択時に「BとCどちらを採用するか」のラジオボタンを表示する
- デフォルトは未選択（強制的に人間が決める）
- 競合セルを未選択のままCSV出力しようとするとエラーメッセージを表示する

### フィルター

- `全て`: 全差分を表示
- `競合のみ`: `conflict: true` の行のみ表示
- `Bの変更`: Bのみが変更している行
- `Cの変更`: Cのみが変更している行

### 一括操作

- 全選択 / 全解除（表示中の行に対して）
- シート単位での一括選択

### CSV出力ボタン

- 選択済み件数を表示（例: `CSV出力（12件選択中）`）
- 競合セルが未解決の場合は出力をブロックしてエラーを表示
- `Blob` を使ってブラウザのダウンロード機能で保存
- UTF-8 BOM付きで出力（VBA・Excelでの文字化け防止）
- カンマ・改行・ダブルクォートを含む値は RFC 4180 準拠でエスケープ

---

## merge_commands.csv 形式

Step 4 VBAとのインターフェース。

```csv
sheet,cell,source,value,type
Sheet1,B4,B,新しい値,value
Sheet1,D7,B,競合を解決した値,value
Sheet2,C10,C,=SUM(A1:A10),formula
Sheet3,,C,図形テキスト内容,shape_text
Sheet3,,B,,shape_add
Sheet3,,C,,shape_delete
```

| カラム | 型 | 内容 |
|--------|-----|------|
| `sheet` | string | シート名 |
| `cell` | string | セル位置（図形の場合は空欄） |
| `source` | `B` or `C` | どちらのファイルからコピーするか |
| `value` | string | コピーする値（VBAの参照用。空欄可） |
| `type` | string | `value` / `formula` / `rich_text` / `comment` / `shape_text` / `shape_add` / `shape_delete` |

→ **CSV形式はStep 4の実装と合わせて最終確定する。**

## 決定事項

| 項目 | 決定内容 |
|------|----------|
| JSX/Babel | 不使用。`React.createElement()` で直接記述。提供元確認未了 + 900KB削減のため排除。 |
| 図形の表示 | セルテーブルとは別セクションに分離。折りたたみ可能。 |
| 図形差分の対象 | 追加・削除のみ。画像・グラフ内容変更は検出しない。 |
| CSV文字コード | UTF-8 BOM付きで出力 |
| CSVエスケープ | RFC 4180 準拠（ダブルクォートで囲み、内部の `"` は `""` に置換） |
| モバイル対応 | 不要（PC業務環境前提） |

## 懸念・未解決事項

### 【高】競合時の選択UIが複雑

B・Cが同じセルを変更している場合のUI（どちらを採用するかの選択）は、
テーブルの行内にラジオボタンを展開する方式が最もシンプルと考えるが、
大量の競合がある場合の操作性が課題。

### 【中】大量差分時のレンダリング性能

差分が数千行ある場合、全行をDOMにレンダリングするとブラウザが重くなる可能性がある。
まず全件レンダリングで実装し、問題が発生した場合にページネーションで対応する。
（仮想スクロールライブラリはインライン埋め込みが困難なため採用しない）

### 【中】図形の `shape_add` / `shape_delete` 時のVBA側の動作

追加された図形をコピーする場合、VBAで図形ごとコピー（`Shape.Copy`）する必要がある。
Step 4 実装時に詳細を確定する。
