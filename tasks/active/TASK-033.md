## 問題

`_compare_cells` はセル座標をキーに1対1で比較するため、
行または列を挿入すると以降の全セルが「差分あり」と誤検知される。

例: Bファイルの3行目に1行挿入 → base の A3 と B の A4 が同じ内容でも
「A3 が削除」「A4 が追加」と判定されてしまう。

## 検討した対処案

### 案A: 行単位 sequence diff（本格対応）
- `difflib.SequenceMatcher` で行リストを揃えてからセル比較
- 外部依存なし（標準ライブラリのみ）
- 行挿入/削除を正確に検出できる
- 列挿入は別途対処が必要
- 実装コスト: 中

### 案B: ヘッダー行をキーに列を揃えてから比較
- 1行目の値で列マッピングを構築
- ヘッダーがない表には使えない
- 実装コスト: 中

### 案C: 差分件数が閾値超えたら警告表示（根本解決ではない）
- 実装コスト: 低

## 現状の判断

- 比較処理全体で約10秒以内に収まる見込みのため、バックエンドで除外するより
  **UIフィルタ（TASK-035）で表示を絞る方針**を優先する
- TASK-035 が実装されれば、行挿入によるズレも画面上で隠せるため、
  本タスクの優先度は低い
- より簡単で実用的な別アプローチを優先するため、本タスクは一旦保留。
  着手前に対処方針を確定すること。
